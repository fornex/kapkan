# Введение

**Kapkan — это бесплатное решение с открытым исходным кодом для обнаружения DDoS-атак (Distributed Denial of Service) и их подавления через [RTBH](/docs/glossary), предназначенное для интернет-провайдеров и хостинг-провайдеров.**

Это единый бинарный файл на Go, который принимает телеметрию потоков — NetFlow v5/v9, IPFIX и sFlow v5 —
от ваших маршрутизаторов, за считаные секунды обнаруживает объёмные (volumetric) атаки на защищаемые вами [префиксы](/docs/glossary) (диапазоны IP
в форме CIDR, например `203.0.113.0/24`) и запускает автоматическое подавление через
[BGP](/docs/glossary) (Border Gateway Protocol — как маршрутизаторы обмениваются маршрутами)
[RTBH](/docs/glossary) (remotely-triggered blackhole — удалённо инициируемая «чёрная дыра»). В комплект входят
REST API, встроенная веб-панель, метрики Prometheus, а также уведомления через Telegram / Slack / электронную почту /
webhook / exec-hook.

Kapkan — это бесплатная замена функциям, за которые коммерческие flow-DDoS-продукты берут деньги.

<Callout type="info" title="Впервые сталкиваетесь с BGP, RTBH или NetFlow?">
Ничего страшного — [Быстрый старт](/docs/quickstart) проведёт вас по всем шагам,
[Глоссарий](/docs/glossary) объясняет каждый термин, а Kapkan не тронет ни одного маршрута, пока
вы сами этого не разрешите. Kapkan работает с *объёмными* атаками (потоками, которые пытаются забить ваш канал или
хост сырым объёмом пакетов и битов), а не с атаками на уровне приложений.
</Callout>

<Callout type="warning" title="Режим dry-run по умолчанию">
Пока вы явно не зададите `dry_run: false`, каждое потенциальное действие blackhole записывается в журнал и доступно
через API, но **никогда не анонсируется** вашим маршрутизаторам. Вы можете запустить Kapkan на боевой
телеметрии и проверить работу обнаружения ещё до того, как он сможет затронуть хотя бы один маршрут. См.
[Модель безопасности](/docs/safety).
</Callout>

## Что он делает

- **Приём (ingest)** — sFlow v5, NetFlow v5/v9 и IPFIX поверх UDP, в библиотечном режиме (без отдельного процесса-спутника),
  посредством [goflow2](https://github.com/netsampler/goflow2).
- **Обнаружение (detect)** — объёмные атаки на каждый адрес назначения с использованием скорректированных по частоте выборки порогов pps / Mbps /
  flows-per-second в скользящем окне, а также опциональных ограничений по протоколам,
  [обучаемых базовых линий](/docs/baselines) и обнаружения исходящих атак.
- **Классификация (classify)** — каждая атака маркируется на основе выборки её потока: amplification
  (NTP / DNS / CLDAP / memcached / SSDP / chargen), SYN / UDP / TCP / ICMP / флуды фрагментами.
- **Подавление (mitigate)** — анонс blackhole-маршрутов `/32` и `/128` через встроенный
  BGP-спикер [GoBGP](https://github.com/osrg/gobgp), отбрасывание только вектора атаки с помощью
  точечных правил [BGP FlowSpec](/docs/flowspec) (RFC 8955/8956) либо перенаправление жертвы в
  [центр очистки трафика](/docs/scrubbing) вместо отбрасывания её трафика — при необходимости поэтапно ужесточаемое со временем
  с помощью [лестниц эскалации](/docs/escalation). Kapkan отслеживает каждое решение о подавлении
  как **блокировку (ban)**; для метода по умолчанию блокировка *и есть* blackhole-маршрут (RTBH).
- **Наблюдение (observe)** — REST API, встроенная веб-панель, Prometheus `/metrics` и
  уведомления через Telegram, Slack, электронную почту, webhook и exec-hook.
- **Хранение (опционально)** — ведение истории атак и трафика в
  [ClickHouse](/docs/storage) для расследований и отчётности.

## Для кого он предназначен

Сетевые операторы — интернет-провайдеры, хостинг-провайдеры и все, кто использует BGP, — которым нужно автоматическое
подавление объёмных атак без коммерческой лицензии (включая пользователей FastNetMon Community,
ищущих альтернативу с открытым исходным кодом).

## Почему Kapkan

| | Kapkan |
| --- | --- |
| **Лицензия** | Apache 2.0, полностью открытый исходный код — включая логику обнаружения |
| **Объём занимаемых ресурсов** | Один статический бинарный файл на Go; ни MongoDB, ни отдельного веб-сервиса |
| **Веб-интерфейс** | Встроенная панель в комплекте, а не платное дополнение |
| **Конфигурация** | Декларативный YAML, который можно хранить в git и сравнивать через diff |
| **Безопасность** | Режим dry-run по умолчанию; TTL, гистерезис и лимиты блокировок заданы на уровне кода |

## Что он пока не делает

Эти возможности есть в плане развития,
но **не** входят в текущий релиз:

- Встроенная дедупликация для высокой доступности между экземплярами
- Захват трафика с зеркалируемого порта (SPAN / AF_PACKET / XDP) как источник приёма
- Маршрутизация уведомлений по арендаторам (сегодня уведомления глобальны; ограничение по арендатору применяется только к [API](/docs/multi-tenancy))
