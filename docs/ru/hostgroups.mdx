# Хост-группы

Единый глобальный блок `thresholds` относится ко всем защищаемым хостам одинаково. На
практике это редко соответствует действительности: клиентский веб-слой ведёт себя
совершенно иначе, чем пул DNS, а сеть, за которой вы хотите лишь наблюдать, ведёт себя
совершенно иначе, чем та, которую нужно автоматически отправлять в blackhole. Хост-группы
позволяют разбить защищаемые `networks` на именованные наборы префиксов, каждый со своими
порогами и политикой митигации, вместо того чтобы настраивать единый набор порогов под
самый «шумный» хост на машине.

Хост-группа — это список префиксов плюс необязательные `thresholds`, `thresholds_outgoing`,
`baseline`, переключатель `ban` и режим `calculation`. Всё, что вы не задали явно,
наследуется из глобальной конфигурации.

<Callout type="info" title="Хост-группы необязательны">
Если блок `hostgroups` отсутствует, каждый адрес назначения внутри `networks` оценивается
по глобальным `thresholds`. Добавляйте группы только там, где подмножеству вашей сети
нужна другая политика.
</Callout>

## Пример

```yaml
networks:
  - "203.0.113.0/24"

thresholds:
  pps: 80000
  mbps: 1000
  flows_per_sec: 35000

hostgroups:
  - name: web                    # tighter per-host limits for this /26
    networks:
      - "203.0.113.0/26"
    thresholds:
      pps: 20000
      mbps: 500
      flows_per_sec: 10000
  - name: customers-no-rtbh      # detect and notify, but never auto-blackhole
    networks:
      - "203.0.113.64/26"
    ban: false
  - name: dns-pool               # alert on the pool's TOTAL traffic
    networks:
      - "203.0.113.128/26"
    calculation: total
    thresholds:
      pps: 300000
      mbps: 4000
      flows_per_sec: 150000
```

Это разбивает защищаемую сеть `203.0.113.0/24` на три политики. Группа `web` понижает планку
для каждого хоста слоя, который никогда не должен быть «шумным». Группа `customers-no-rtbh`
по-прежнему обнаруживает атаки и уведомляет о них, но оставляет митигацию на ваше
усмотрение. Группа `dns-pool` перестаёт смотреть на отдельные резолверы и вместо этого
следит за совокупной скоростью трафика пула.

## Правила

- **Принадлежность по совпадению самого длинного префикса (longest-prefix-match).** Каждый
  хост принадлежит ровно одной группе — той, у которой совпавший префикс наиболее
  специфичен (длиннее). Хост, попадающий одновременно в группу `/24` и во вложенную в неё
  группу `/26`, относится к `/26`.
- **Неявный глобальный резерв (fallback).** Хосты, не совпавшие ни с одной группой,
  попадают в неявную группу `global`, которая несёт верхнеуровневые `thresholds` (а также
  `thresholds_outgoing` / `baseline`, если они заданы). Вам никогда не нужно перечислять
  каждый префикс — всё, что осталось, использует глобальную политику.
- **Префиксы группы должны лежать внутри `networks`.** Группа может лишь сузить защиту,
  которую вы уже объявили; префикс за пределами ваших защищаемых `networks` — это ошибка
  конфигурации.
- **Пороги необязательны, но действуют по принципу «всё или ничего».** Опустите
  `thresholds` — и группа унаследует глобальные пороги. Чтобы переопределить их для этой
  группы, задайте все три ключа — `pps`, `mbps` и `flows_per_sec`: частичный блок
  отвергается. (`thresholds_outgoing` устроен иначе: там достаточно хотя бы одного ключа.)
- **`ban: false` отключает авто-RTBH.** Хосты группы по-прежнему обнаруживаются,
  классифицируются и для них отправляются уведомления, но автоматический blackhole не
  анонсируется. Ручные баны через API по-прежнему работают и по-прежнему применяются к этим
  хостам.
- **`calculation: total` суммирует группу.** Вместо оценки каждого хоста совокупный трафик
  группы сравнивается с порогами. Группы с суммарным расчётом сообщают `scope: "group"` в
  событиях, уведомлениях и API и **никогда не банят автоматически** — нет единственного
  хоста, который можно было бы отправить в blackhole. Значение по умолчанию,
  `calculation: per_host`, оценивает каждый хост независимо.
- **Хост-группы перечитываются «на горячую» (hot-reload)** вместе с остальной
  конфигурацией (`SIGHUP` или `POST /api/v1/config/reload`); чтобы добавить, удалить или
  перенастроить группу, перезапуск не требуется.

<Callout type="warning" title="Суммарные группы никогда не отправляют в blackhole автоматически">
Группа с `calculation: total` фиксирует атаку для набора префиксов в целом, поэтому нет
единственного адреса, для которого можно было бы анонсировать `/32` или `/128`. Указание
`ban: true` для суммарной группы не игнорируется молча — это приводит к ошибке конфигурации
при запуске и перезагрузке. Используйте суммарные группы для оповещения о совокупной
нагрузке (anycast или пул DNS); сочетайте их с отдельной группой по каждому хосту или
ручными банами, если вам также нужна автоматическая митигация.
</Callout>

## Переопределения на уровне группы

Помимо `thresholds`, каждая группа может также нести свои собственные исходящие пороги и
baseline или отказаться от функции, включённой в глобальной конфигурации.

- **`thresholds_outgoing`** — задаёт группе собственную политику для атак, исходящих от её
  хостов (скомпрометированные машины, атакующие наружу). Использует те же ключи, что и
  `thresholds`; должен быть задан хотя бы один. См. [Обнаружение](/docs/detection) о
  детектировании исходящего трафика.
- **`baseline`** — группа либо наследует глобальный блок обучаемого baseline, либо
  переопределяет его целиком собственным. Чтобы полностью исключить группу из baseline,
  задайте `baseline: { enabled: false }` — тогда группа использует только статические
  пороги.
- **`tenant`** — помечает владельца группы, чтобы API-токен с областью видимости арендатора
  (tenant) видел только атаки, баны и хосты своих собственных групп. См.
  [Мультиарендность](/docs/multi-tenancy).

```yaml
hostgroups:
  - name: edge
    networks:
      - "203.0.113.192/26"
    thresholds:
      pps: 40000
      mbps: 800
      flows_per_sec: 20000
    thresholds_outgoing:        # watch for compromised hosts in this group
      pps: 30000
      udp_pps: 15000
    baseline:
      enabled: false            # static thresholds only, no learning
```

Переопределения на уровне группы независимы: группа может задать `thresholds`, но
унаследовать глобальный `baseline`, либо задать собственный `baseline`, унаследовав
глобальные `thresholds`. Всё, что осталось незаданным, возвращается к глобальному значению.
