# Глоссарий

Однострочные определения терминов, которые остальная документация считает известными. Если на какой-то странице встретилось незнакомое слово — скорее всего, оно есть здесь. Определения намеренно практичные, а не точные по RFC.

## Атаки

| Термин | Значение |
| --- | --- |
| **DDoS** | Distributed Denial of Service (распределённый отказ в обслуживании) — множество источников одновременно заваливают цель трафиком, чтобы вывести её из строя. |
| **Объёмная атака** | Поток, который побеждает за счёт чистого объёма (пакетов или бит в секунду), а не за счёт хитрого злоупотребления протоколом. Именно это обнаруживает Kapkan. |
| **Усиление (amplification)** | Атакующий подменяет адрес жертвы и задаёт стороннему сервису (NTP, DNS, CLDAP, memcached, SSDP, chargen) маленький вопрос, на который приходит огромный ответ — так небольшая полоса атакующего превращается в большую у жертвы. |
| **SYN / UDP / ICMP флуд** | Потоки из пакетов одного типа, названные по типу пакета. SYN-флуд исчерпывает состояние соединений; UDP/ICMP-флуд просто топит канал. |
| **Ковровая бомбардировка (carpet bomb)** | Атака, тонко размазанная сразу по множеству адресов в префиксе, так что ни один отдельный хост не выглядит достаточно плохо, чтобы превысить порог по одному хосту. У Kapkan для этого есть отдельное обнаружение размазывания по подсети. |
| **Вектор** | Конкретная форма атаки (например, «усиление NTP», «SYN-флуд»). FlowSpec может отбрасывать один вектор, не трогая остальной трафик хоста. |

## Телеметрия потоков

| Термин | Значение |
| --- | --- |
| **Поток (flow)** | Сводная запись об обмене между двумя адресами (кто, кому, какой протокол, сколько пакетов/байт) — её экспортирует ваш маршрутизатор вместо копии каждого пакета. |
| **NetFlow / IPFIX** | Протокол экспорта потоков от Cisco и его стандартизированный IETF-преемник. Kapkan слушает оба на UDP `:2055`. |
| **sFlow** | Протокол экспорта на основе выборки пакетов. Kapkan слушает sFlow v5 на UDP `:6343`. |
| **Частота выборки (sampling rate)** | Маршрутизаторы экспортируют только 1 пакет из *N*, чтобы сберечь ресурсы. Kapkan умножает счётчики обратно на *N*, чтобы оценить реальный трафик, поэтому это число должно быть верным — иначе обнаружение будет неправильным. |

## BGP и маршрутизация

| Термин | Значение |
| --- | --- |
| **BGP** | Border Gateway Protocol — то, как маршрутизаторы сообщают друг другу, какие диапазоны IP они могут достичь. Kapkan выступает как BGP-говорящий, который указывает вашим маршрутизаторам установить (или снять) blackhole-маршруты. |
| **ASN** | Autonomous System Number — числовой идентификатор сети в интернете (например, вашего провайдера). Сессии BGP устанавливаются между двумя ASN. |
| **eBGP / iBGP** | BGP между *разными* ASN (внешний) против BGP *внутри* вашего собственного ASN (внутренний). Некоторые атрибуты (например, LOCAL_PREF) передаются только по iBGP. |
| **Префикс / CIDR** | Диапазон IP, записанный как `адрес/длина`, например `203.0.113.0/24` (256 адресов) или одиночный хост `203.0.113.5/32`. «CIDR» — это просто такая запись. |
| **Next-hop** | Адрес, на который маршрутизатор должен отправлять подходящий трафик. |
| **Discard next-hop** | Next-hop, который ваши маршрутизаторы настроены отбрасывать. RTBH работает за счёт анонсирования маршрута жертвы с таким next-hop, и маршрутизатор выбрасывает трафик. |
| **Community** | Метка, прикреплённая к BGP-маршруту. Вышестоящие провайдеры используют согласованное «blackhole community» в значении «пожалуйста, отправьте это в null-маршрут за меня». |
| **LOCAL_PREF** | Значение предпочтения BGP, которое имеет смысл только внутри вашей собственной сети (iBGP). |
| **NLRI** | Network Layer Reachability Information — техническое название для «анонсируемого маршрута» (префикса). |
| **AFI / SAFI** | Метки семейств адресов, которыми BGP пользуется, чтобы держать маршруты IPv4, IPv6 и FlowSpec в раздельных каналах одной сессии. |
| **Graceful Restart / LLGR** | Функции BGP, которые сохраняют маршруты соседа на месте, пока сессия ненадолго перезапускается, чтобы перезапуск Kapkan не вызывал флаппинг ваших blackhole. LLGR — это «долгоживущий» вариант. |
| **End-of-RIB** | Маркер, который BGP-говорящий отправляет, чтобы сказать: «Я закончил отправлять вам свои маршруты». |
| **uRPF** | Unicast Reverse Path Forwarding — функция маршрутизатора, которая отбрасывает пакеты, чей *исходный* адрес выглядит поддельным. В паре с RTBH она позволяет blackhole также ловить трафик *от* заблокированного префикса. |
| **Anycast** | Один и тот же адрес, анонсируемый из множества мест; сеть доставляет каждого пользователя к ближайшему. |

## Методы подавления атак

| Термин | Значение |
| --- | --- |
| **RTBH / blackhole** | Remotely-Triggered Black Hole — анонс `/32` (или `/128`) жертвы с discard next-hop, чтобы маршрутизаторы отбрасывали *весь* трафик к ней. Грубо, но мгновенно. Режим Kapkan по умолчанию. |
| **FlowSpec** | Расширение BGP (RFC 8955/8956), которое анонсирует *фильтр* — протокол, порты, размер пакета — чтобы маршрутизаторы отбрасывали только трафик атаки, оставляя остальной хост доступным. |
| **Очистка / перенаправление (scrubbing / divert)** | Вместо отбрасывания трафика жертвы его перенаправляют через «центр очистки», который отфильтровывает атаку и возвращает чистый трафик. Требует сервиса очистки, который вы уже эксплуатируете или покупаете. |

## Понятия Kapkan

| Термин | Значение |
| --- | --- |
| **Блокировка (ban)** | Запись Kapkan об одном решении по подавлению атаки против одной цели — независимо от того, анонсируется ли оно как blackhole, правило FlowSpec или перенаправление. |
| **Dry-run** | Режим Kapkan по умолчанию: обнаруживать и записывать всё, но никогда не анонсировать маршрут. Позволяет безопасно проверить работу на реальном трафике. |
| **TTL** | Time To Live — как долго длится блокировка, прежде чем её автоматически снимут. Постоянных блокировок не бывает. |
| **Гистерезис** | Задержка перед снятием блокировки после спада трафика, чтобы блокировка не мигала туда-сюда на грани атаки. |
| **Белый список (whitelist)** | `protected_whitelist` — адреса, которые нельзя заблокировать ни обнаружением, ни вручную. |
| **Hostgroup** | Именованный набор префиксов со своими порогами и политикой подавления атак, чтобы можно было защищать (скажем) игровые серверы иначе, чем веб-серверы. |
| **Базовая линия (baseline)** | Выученный «нормальный» уровень трафика для хоста (с помощью EWMA — скользящего среднего, которое сильнее взвешивает недавний трафик), чтобы пороги могли адаптироваться, а не быть фиксированными. |
| **Порог (threshold)** | Уровень трафика (pps / Мбит/с / потоков в секунду), выше которого Kapkan объявляет атаку. |
