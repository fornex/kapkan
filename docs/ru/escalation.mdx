# Лестницы эскалации

Одиночный метод `mitigation` запускает один и тот же ответ в момент обнаружения атаки.
Лестница эскалации вместо этого **усиливает ответ по мере того, как атака затягивается** —
декларативно, без скриптов-обработчиков (callback).

Большинству развёртываний достаточно одного метода `mitigation:`. Лестница нужна только тогда,
когда вы хотите сначала ответить мягче (например, оповещение, затем точечная фильтрация),
прежде чем прибегать к полному [blackhole](/docs/glossary).

```yaml
escalation:                         # supersedes `mitigation` when present
  - { after_seconds: 0,   action: none }       # alert only at first
  - { after_seconds: 30,  action: flowspec }   # still under attack after 30s → surgical drop
  - { after_seconds: 90,  action: divert }     # still under attack after 90s → scrub
  - { after_seconds: 300, action: blackhole }  # still under attack after 300s → blackhole
flowspec:
  action: discard
scrubbing:
  next_hop: "192.0.2.100"   # scrubbing center; see Traffic diversion
  community: "65000:200"
```

[Блок `bgp`](/docs/mitigation#конфигурация-bgp) обязателен, чтобы любая ступень могла
анонсироваться; блоки `flowspec` и `scrubbing` выше нужны только для ступеней `flowspec`
и `divert`.

## Как применяются ступени

Значение `after_seconds` каждой ступени отсчитывается от начала атаки. Ступень применяется,
как только истекло соответствующее время **и блокировка всё ещё активна**. (Блокировка отзывается только
после того, как трафик остаётся ниже порога в течение настроенного периода остывания,
`ban.unban_hysteresis_seconds` — см. [Справочник конфигурации](/docs/configuration).) Часы
отсчёта ступеней при этом непрерывно идут от начала атаки.

- Первая ступень должна быть на `0` секундах.
- Лестница может иметь не более **5 ступеней** — это жёсткий предел; демон откажется
  запускаться с более длинной лестницей.
- `action` — это `none` (только оповещение), `flowspec`, `divert` ([scrubbing](/docs/scrubbing))
  или `blackhole`.
- Лестница может только **удерживать или усиливать** ответ: `none` < `flowspec` < `divert` <
  `blackhole`. Деэскалация между ступенями является ошибкой конфигурации.

## Make-before-break

Переход на новую ступень выполняется по принципу make-before-break (сначала установить,
потом разорвать): новая ступень анонсируется первой, и предыдущая отзывается только после
того, как это удалось, — поэтому жертва ни на мгновение не остаётся без защиты в момент
переключения. Если анонс не удаётся, блокировка удерживает рабочую ступень и повторяет попытку
на следующем такте. Если сразу подходит срок нескольких ступеней (длительная атака или
демон навёрстывает отставание после паузы), блокировка сразу прыгает на самую высокую подошедшую
по сроку ступень и никогда не анонсирует пропущенные ступени.

Ступень `flowspec` или `divert`, анонс которой отклонил вышестоящий пир, откатывается к
blackhole (если вы не задали `ban.fallback: none` в [Справочнике конфигурации](/docs/configuration));
API и уведомления показывают это через поле `fell_back_from`. У ступеней blackhole и
«только оповещение» запасного варианта нет.

Единственное исключение из последовательности «анонс, затем отзыв» — переход
`divert → blackhole`: оба анонсируют один и тот же host-route (адрес жертвы `/32` или `/128`),
поэтому анонс blackhole атомарно заменяет маршрут divert — без отдельного отзыва, без разрыва.
Переключения между семействами (правила FlowSpec против host-route) следуют описанному выше
случаю make-before-break.

## Жизненный цикл и совместимость

Лестница **переопределяема для каждой hostgroup** и разделяет остальной жизненный цикл блокировки:
автоматический отзыв по TTL, ограничение `max_active_bans`, ограничения радиуса поражения
(`max_banned_fraction`, `max_bans_per_window`), абсолютный белый список и dry-run (который
продвигает лестницу и логирует каждую ступень, но никогда не анонсирует).

Когда блок `escalation` не задан, одиночный метод `mitigation` ведёт себя ровно как лестница
из одной ступени на `0` секундах — полная обратная совместимость. Каждая блокировка в `/api/v1/bans`
содержит всю настроенную лестницу целиком (`escalation`), индекс текущей ступени
(`escalation_step`) и метод активной ступени (`method`).
