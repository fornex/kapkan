# Быстрый старт

Это руководство проведёт вас от свежего клона репозитория до работающего экземпляра Kapkan в режиме dry-run, который обнаруживает
атаки на тестовый префикс. Ничто из описанного здесь не способно анонсировать маршрут — режим dry-run включён по умолчанию.

<Callout type="info" title="Предварительные требования">
Go 1.22 или новее. Чтобы видеть обнаружения в реальном времени, вам также понадобится маршрутизатор, экспортирующий NetFlow / IPFIX /
sFlow, — но вы можете проверить работу сквозного сценария и без него, используя встроенный генератор flow (см.
последний раздел).
</Callout>

## 1. Сборка бинарника

```sh
make build
```

В результате создаётся единый статический бинарник `kapkan`.

## 2. Запуск в режиме dry-run

Запустите Kapkan со встроенной конфигурацией для разработки и логами в читаемом виде:

```sh
make run-dev
# equivalent to:
./kapkan -config configs/dev.yaml -log-format text
```

Конфигурация для разработки ([`configs/dev.yaml`](/docs/configuration)) содержит `dry_run: true`, тестовую
защищаемую сеть `203.0.113.0/24` и слушает sFlow на `:6343`, а
NetFlow/IPFIX — на `:2055`.

## 3. Направьте экспортёры на Kapkan

Настройте flow-экспортёры ваших маршрутизаторов на отправку данных на порты, которые слушает Kapkan:

| Протокол | Порт по умолчанию |
| --- | --- |
| sFlow v5 | `:6343` |
| NetFlow v5/v9 + IPFIX | `:2055` |

NetFlow и IPFIX используют один и тот же UDP-сокет. Должен быть настроен хотя бы один слушатель.

## 4. Понаблюдайте за работой

По умолчанию Kapkan отдаёт свой REST API и метрики на `127.0.0.1:8080`:

```sh
# Current mode, protected networks, thresholds, active attack/ban counts
curl -s localhost:8080/api/v1/status | jq

# Active attacks plus the last 100 that ended, with samples and classification
curl -s localhost:8080/api/v1/attacks | jq

# Prometheus metrics
curl -s localhost:8080/metrics | grep kapkan_
```

Также можно открыть встроенную [панель управления](/docs/dashboard) по адресу
[http://localhost:8080](http://localhost:8080) в браузере.

## 5. Нет маршрутизатора? Сгенерируйте синтетические атаки

В состав Kapkan входит `pkg/flowgen`, который формирует реальные датаграммы NetFlow / sFlow в проводном формате —
включая паттерны amplification, SYN-flood и UDP-flood. Он используется на протяжении всего набора тестов
для проверки сквозного обнаружения без обращения к реальному маршрутизатору. Интеграционный тест
в `internal/app` воспроизводит флуд типа NTP-amplification через реальный UDP-сокет, направляя его на экземпляр в режиме
dry-run, и убеждается, что атака и её (автоматически истекающая) виртуальная блокировка появляются в
API.
