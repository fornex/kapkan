# Базовые уровни

С помощью блока `baseline` Kapkan непрерывно изучает нормальный уровень трафика каждого хоста
и подстраивает под него детектирование. Он поддерживает EWMA (экспоненциально взвешенное
скользящее среднее) реальной, скорректированной на сэмплирование скорости **для каждого хоста
и каждого направления** — а для hostgroup с `calculation: total` ещё и суммарное значение по
группе — после чего выводит эффективный порог `learned_normal × factor`. Хост, который обычно
даёт 10k pps, помечается примерно на 30k вместо ожидания глобальных 80k.

Это режим «перестань настраивать пороги вручную». Если автоматический baseline в FastNetMon —
это офлайн-калькулятор, который вы запускаете и из которого копируете цифры, то у Kapkan он
онлайновый: он непрерывно следует за вашим трафиком и пересчитывает эффективный порог по мере
того, как нормальный уровень хоста дрейфует.

<Callout type="info" title="Базовые уровни ужесточают, но никогда не ослабляют">
Ваши статические [пороги](/docs/detection) всегда остаются в силе как жёсткий потолок. Базовый
уровень может только сделать детектирование *более* чувствительным для хоста — он никогда не
может поднять планку выше того, что вы настроили.
</Callout>

## Пример

```yaml
baseline:
  factor: 3              # attack = traffic above learned_normal × factor
  half_life_seconds: 3600
  warmup_seconds: 600
  floor:
    pps: 5000
    mbps: 50
    flows_per_sec: 2000
```

Блок необязателен. При его отсутствии Kapkan использует только ваши статические пороги.
Эффективный порог для каждого хоста и метрики таков:

```text
effective = clamp(learned_normal × factor, low = floor, high = static_threshold)
```

Словами: `learned_normal × factor`, ограниченное снизу значением `floor`, а сверху —
статическими порогами (потолком).

## Ключи

| Ключ | По умолчанию | Значение |
| --- | --- | --- |
| `factor` | `3` | Множитель к обученному нормальному уровню. Эффективный порог равен `learned_normal × factor`. |
| `half_life_seconds` | `3600` | Период полураспада EWMA. Время, за которое старое наблюдение теряет половину своего веса; большие значения делают базовый уровень более медленным и стабильным. |
| `warmup_seconds` | `600` | Свежеобнаруженный хост в течение этого времени ограничивается только статическими порогами, отсчёт ведётся от его первого реального трафика. |
| `floor.pps` / `floor.mbps` / `floor.flows_per_sec` | пример `5000` / `50` / `2000` | Нижняя граница эффективного порога по каждой метрике, чтобы тихие хосты никогда не становились сверхчувствительными к малейшему всплеску. |

## Конструкция с защитой от отравления

Наивный обучаемый базовый уровень уязвим: подайте на хост медленно нарастающий поток — и он
выучит этот поток как «нормальный», поднимая планку до тех пор, пока реальные атаки не начнут
проскальзывать под ней. Базовый уровень Kapkan построен так, чтобы этому сопротивляться.
Статические пороги остаются в качестве защиты, а несколько правил ограничивают то, что базовый
уровень вообще способен сделать.

- **Потолок.** Трафик выше статических порогов всегда срабатывает. Отравленный или
  быстро выросший базовый уровень никогда не может поднять эффективный порог выше того, что вы
  настроили.
- **Пол.** Эффективный порог никогда не опускается ниже `floor`, поэтому обычно тихий хост не
  становится сверхчувствительным, срабатывающим на любой небольшой всплеск.
- **Заморозка во время атаки.** Пока хост находится под атакой — включая хвост
  `unban_hysteresis_seconds` после спада скорости — его базовый уровень вообще не обучается.
  Трафик атаки никогда не учит хост тому, что атака — это норма.
- **Ограниченное обучение.** Вне атак каждый обучающий сэмпл ограничивается значением
  `baseline × factor` перед обновлением EWMA. Поэтому медленно нарастающий атакующий может
  поднять базовый уровень максимум на `2^(factor−1)` за период полураспада — около `4×` в час
  при значениях по умолчанию (`factor: 3`, `half_life_seconds: 3600`). Это часы подъёма от
  нормального уровня до статического потолка и никогда выше него. Агрессивные настройки
  (большой `factor` или малый `half_life_seconds`) сужают это окно, поэтому выбирайте их
  осознанно.
- **Обучение только на реальном трафике.** Направление без трафика в окне никогда не обучает
  свой базовый уровень. Хост только с входящим трафиком сохраняет свой статический исходящий
  порог, а пустая группа `total` никогда не «прогревается» до нулевого базового уровня.
- **Прогрев.** Свежеобнаруженный хост защищён статическими порогами только в течение
  `warmup_seconds`, отсчёт ведётся от его первого реального трафика. Вытесненный (давно тихий)
  хост заново проходит прогрев, когда возвращается.

<Callout type="warning" title="Хост, уже подвергающийся флуду при первом обнаружении">
Сам трафик прогрева обучает начальный базовый уровень. Хост, который *уже* находится под
суб-статическим флудом в тот момент, когда Kapkan впервые его видит, выучит этот флуд как
«нормальный» — ограниченный статическим потолком, но для хоста, атакуемого с момента первого
обнаружения, чистого эталона нет. Установите `warmup_seconds` хотя бы в несколько раз больше
`half_life_seconds`, чтобы базовый уровень сошёлся прежде, чем начнёт что-либо ограничивать.
</Callout>

## Видимость

Обученные уровни доступны для каждого хоста через [REST API](/docs/api). Снимок
`GET /api/v1/hosts` содержит `baseline` (входящий) и `baseline_out` (исходящий) рядом с
измеренными скоростями каждого хоста по направлениям и состоянием атаки.
[Дашборд](/docs/dashboard) отображает те же обученные базовые уровни рядом с топовыми
источниками трафика, так что вы можете видеть, к какому уровню привязан каждый хост.

## По hostgroup

Базовые уровни следуют области действия [hostgroup](/docs/hostgroups). Группа либо наследует
глобальный блок `baseline`, либо переопределяет его целиком — частичного слияния нет. Чтобы
полностью исключить группу из базовых уровней, сохранив глобальный блок для всех остальных,
отключите его на группе:

```yaml
hostgroups:
  - name: dns-pool
    networks: ["203.0.113.128/26"]
    calculation: total
    baseline: { enabled: false }   # this group uses static thresholds only
```

Для групп с `calculation: total` базовый уровень обучает суммарный трафик группы, а не уровни
отдельных хостов, и те же правила защиты от отравления применяются к этой сумме.
